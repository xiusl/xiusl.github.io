部署自己的 sentry 服务

自建 sentry 服务，作为错误信息监控。采用的是旧的 9.1.2 版本，如需最新版本，参考官方文档。

推荐在 Debain 系统上进行部署，之前使用了 REHL 系列的系统，self-hosted-912-web 服务会分配和占用很大的内存，具体什么原因还不清楚。

主机上需要安装好 docker 服务

在 github 上下载官方提供的部署脚本（附：下载地址），解压文件。

由于目前安装的 docker compose 是最新的版本，需要修改 install.sh 中的部分代码，主要是替换 docker-compose 为 docker compose，完成代码参考 install.sh

执行修改后的 install.sh 脚本

脚本执行过程中会提示是否创建 sentry 账号，可以选择在这个地方创建

脚本执行完成后，启动服务

查询服务状态，正常会有以下 7 个服务运行

通过网页打开 http://your-host-ip:9000 登录创建好的账号就可以使用了

可以在前面加一层 nginx 代理，实现自定义域名和 https 访问
